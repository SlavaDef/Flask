Тег `<nav>` в HTML є семантичним елементом, який використовується для визначення розділу навігації на
веб-сторінці. Ось основні характеристики та призначення цього тегу:
1.Основне призначення:
    - Визначає секцію з навігаційними посиланнями
    - Може містити меню, списки посилань, навігаційні панелі

2.Семантичне значення:
    - Допомагає пошуковим системам краще розуміти структуру сайту
    - Покращує доступність сайту для програм зчитування з екрану

3.Приклади використання:
    - Головне меню сайту
    - Навігаційні ланцюжки (breadcrumbs)
    - Бічна навігація
    - Нижнє меню (футер)

У цьому коді тег `<nav>` використовується для створення навігаційного ланцюжка (breadcrumbs)

Атрибути `id` та `name` в HTML мають різні призначення:
1. `id`:
    - Унікальний ідентифікатор елемента на всій сторінці
    - Використовується для:
        - JavaScript маніпуляцій (`document.getElementById()`)
        - CSS стилізації (`#to_currency { ... }`)
        - Створення зв'язків з `<label>` через `for="id"`

    - Повинен бути унікальним на сторінці
    - Не передається на сервер при відправці форми

2. `name`:
    - Використовується для ідентифікації полів форми
    - Передається на сервер при відправці форми
    - Може повторюватися (наприклад, для radio buttons)
    - Визначає ключ у даних форми
-----------------------------------------------------------------------------------------------------------------------
Тег `<div>` (division - розділ) - це один з найбільш універсальних блочних елементів HTML. Ось його основні характеристики та призначення:
1. **Основне призначення**:
    - Створення логічних розділів контенту
    - Групування елементів
    - Контейнер для інших елементів

2. **Особливості**:
    - Блочний елемент (починається з нового рядка)
    - За замовчуванням не має візуального оформлення
    - Займає всю доступну ширину батьківського елементу
**Важливо**: Хоча `<div>` дуже універсальний, краще використовувати семантичні теги (`<header>`, <nav>`,
`<main>`, `<article>`, `<section>`, `<footer>`) там, де це можливо, для кращої доступності та SEO
